## 시작하며

기계공학 전공에, 지금은 용접 벨로우즈랑 반도체 장비 부품 만드는 회사에서 금형 설계랑 설비 관리를 하고 있다. 코딩이랑은 거리가 먼 커리어였는데, 최근에 "AI 도구로 뭔가 만들어보자"는 생각으로 사이드 프로젝트를 시작했다.

목표는 단순했다. **클로드를 그냥 질문-답변 챗봇이 아니라, 실제로 파일을 읽고 쓰고 코드를 짜는 비서로 만들기.** 이걸 가능하게 해주는 게 MCP(Model Context Protocol)라는 걸 알게 됐고, 오늘은 그 첫 세팅기를 기록해본다.

## 왜 24개 중에 6개만 골랐나

MCP 서버 종류를 찾아보니 생각보다 많았다. Chrome 자동화, DB 연결, 협업툴 연동까지 24개 후보를 놓고 봤는데, 전부 다 깔면 오히려 뭐가 뭔지 헷갈릴 것 같았다.

그래서 지금 단계(콘텐츠 제작 + 앱개발 초기)에 진짜 필요한 것만 추렸다.

1. **Claude Code** — 파일 읽기/쓰기, 터미널 실행까지 다 되는 핵심 도구
2. **Filesystem** — 로컬 프로젝트 폴더 직접 접근
3. **Git** — 로컬 버전관리
4. **GitHub** — 원격 저장소, 이슈 관리
5. **Memory** — 세션 넘어가도 프로젝트 맥락 기억
6. **SQLite** — 가벼운 로컬 DB (아직 서버급 DB는 오버스펙이라 판단)

Slack, Notion, Postgres, Docker 같은 건 나중에 협업하거나 앱이 커지면 그때 추가하기로 했다.

## 첫 번째 삽질: GitHub 토큰 권한

> 여기에 실제로 PAT(Personal Access Token) 발급하면서 헷갈렸던 부분, 스코프 설정 실수담 기록

## 두 번째 삽질: Filesystem 접근 범위

> 여기에 폴더 권한을 너무 넓게 잡았다가 다시 좁힌 이야기, 혹은 반대로 너무 좁게 잡아서 삽질한 이야기 기록

## 지금까지 정한 앱개발 스택

MCP 세팅에 맞춰서 스택도 같이 정했다.

- **Next.js (App Router)** + TypeScript
- **Tailwind CSS**
- **SQLite + Prisma**
- 블로그는 **MDX 파일**로 `content/posts`에 직접 관리 (지금 이 글도 그렇게 쓰는 중)

왜 이 조합인지, 설치한 MCP 도구들이랑 어떻게 맞물리는지는 다음 편에서 더 자세히 풀어볼 예정.

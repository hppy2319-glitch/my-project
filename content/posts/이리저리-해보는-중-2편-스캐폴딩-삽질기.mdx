## 시작하며

1편에서 MCP 세팅까지 끝내고, 이번엔 실제로 Next.js 프로젝트를 스캐폴딩하면서 겪은 일들을 기록한다. 계획대로 안 되는 게 개발이라더니, 진짜였다.

## 삽질 1: Prisma 대신 Drizzle을 쓰게 된 사연

원래 계획은 **Prisma + SQLite**였다. Prisma가 유명하고 문서도 많으니까 당연히 될 줄 알았다.

```
npx prisma init --datasource-provider sqlite
```

근데 이 명령어를 실행하는 환경(AI가 작업하던 샌드박스)에서 에러가 났다. Prisma가 스키마 엔진 바이너리를 외부 서버에서 받아오려고 하는데, 그 도메인 자체가 막혀 있었던 것.

```
Error: Failed to fetch sha256 checksum...403 Forbidden
```

여기서 배운 것: **Prisma는 편하지만 내부적으로 별도 엔진 바이너리를 다운로드받는 구조**라서, 네트워크 제약이 있는 환경에서는 막힐 수 있다. 대신 순수 JS/TS로만 동작하는 **Drizzle ORM + better-sqlite3** 조합으로 바꿨더니 바로 됐다.

교훈: 라이브러리를 고를 때 "이게 실행 중에 뭘 추가로 다운로드받는가"도 생각해봐야 한다는 걸 처음 알았다.

## 삽질 2: 구글 폰트도 막힐 수 있다

Next.js 기본 템플릿은 `next/font/google`로 Geist 폰트를 자동으로 불러온다. 근데 이것도 `fonts.googleapis.com`에 접근해야 하는 구조라, 같은 이유로 빌드가 안 됐다.

결국 폰트 코드를 걷어내고 시스템 기본 폰트로 임시로 돌렸다. (내 컴퓨터에서는 문제없이 될 부분이라 나중에 되살릴 예정)

## 삽질 3: Git을 설치 안 하고 명령어부터 쳤다

GitHub에 코드를 올리려고 터미널을 처음 열었는데,

```
'git'은(는) 내부 또는 외부 명령... 아닙니다
```

이 메시지를 보고서야 "아, Git을 설치를 안 했지"를 깨달았다. [git-scm.com](https://git-scm.com)에서 설치하고, **터미널을 완전히 새로 열어야** 인식된다는 것도 이번에 처음 알았다. (설치 전에 열어둔 창은 계속 인식을 못 한다.)

## 삽질 4 (feat. 진땀): PAT 토큰을 캡처해버림

GitHub에 push하려면 Personal Access Token이 필요한데, 발급받은 토큰 값을 캡처해서 공유해버렸다. 스크린샷에 토큰 문자열이 그대로 찍힌 걸 뒤늦게 알아채고 바로 삭제 후 재발급했다.

**토큰은 비밀번호랑 똑같이 취급해야 한다**는 걸 몸으로 배운 순간. 다행히 "이 저장소 하나만, Contents 권한만" 이렇게 좁게 잡아뒀어서 최악의 상황은 피했다.

## 오늘의 교훈 정리

1. 라이브러리 선택 시 "실행 중 추가 다운로드가 있는가"도 체크
2. 설치 후엔 터미널을 새로 열자
3. 토큰/키는 절대 캡처해서 공유하지 말 것 — 노출되면 바로 재발급
